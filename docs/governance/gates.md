# Gate 清单（G0-G7）

> 目标：把“口头完成”变成“证据完成”。
> 规则：任何 Gate 失败，流程立即停止并回到最近可验证状态。

## G0 会话启动门禁
- 输入：`docs/templates/handoff.md` 已填写版本
- 检查器：handoff 字段完整性检查（会话、目标、证据索引、回滚点）
- 通过条件：必填项完整且“本轮唯一目标”清晰
- 失败动作：禁止开始执行，只允许补全 handoff

## G1 规划审查门禁
- 输入：本轮一步计划 + PRD 条款引用
- 检查器：条款映射检查、单步粒度检查、接口/类型先行检查
- 通过条件：每一步都有“输入/输出/完成判定”
- 失败动作：退回重拆，不允许写实现

## G2 Red 证据门禁
- 输入：失败测试输出日志
- 检查器：失败原因检查（断言失败优先，不接受环境报错冒充 Red）
- 通过条件：测试失败可复现且与目标功能相关
- 失败动作：禁止进入实现，先修测试场景

## G3 Green 证据门禁
- 输入：实现变更 + 测试通过日志
- 检查器：目标测试通过、无新增回归、变更范围匹配计划
- 通过条件：相关测试全绿，且未越界改动
- 失败动作：回退到 G2

## G4 重构门禁
- 输入：重构后 diff + 测试结果
- 检查器：行为不变检查（测试不变绿）、职责边界检查
- 通过条件：测试仍全绿，且未引入职责漂移
- 失败动作：回滚本次重构

## G5 文档一致性门禁
- 输入：本轮文档变更
- 检查器：`tools/scripts/check_docs_consistency.ps1`
- 通过条件：通过一致性脚本；新增文档仅引用归属规则，不复制权威正文
- 失败动作：阻断提交，修正文档后重试

## G6 提交门禁
- 输入：本次提交内容
- 检查器：`tools/githooks/pre-commit` + `tools/githooks/commit-msg`
- 通过条件：本地门禁全部通过，提交信息格式合规
- 失败动作：阻断 commit

## G7 合并门禁
- 输入：PR / 合并请求
- 检查器：CI（镜像本地 hooks 检查集）+ 全量测试 + 审计结论
- 通过条件：CI 全通过且审计无阻断项
- 失败动作：禁止合并

---

## 统一失败处理
1. 停止当前执行
2. 记录失败 Gate 与失败证据
3. 回到最近可验证状态
4. 修复后从失败 Gate 重新进入
