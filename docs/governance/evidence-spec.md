# 证据规范（计划 / 执行 / 验证）

> 目标：拒绝“口头完成”，统一以证据判定完成度。

## 1) 证据类型

### A. 计划证据
- 必须包含：
  - PRD 条款引用
  - 本步目标（单步）
  - 输入/输出定义
  - 完成判定语句
- 典型载体：计划文档片段、任务卡

### B. 执行证据
- 必须包含：
  - 文件变更清单（新增/修改/删除）
  - 命令输出（关键命令）
  - 测试日志（尤其 Red/Green）
- 典型载体：diff 摘要、命令日志、测试报告

### C. 验证证据
- 必须包含：
  - 独立复核结果（审计视角）
  - Gate 通过/失败记录
  - 失败时回滚点
- 典型载体：审计报告、门禁日志、复跑结果

## 2) 存储路径与命名
- 存储目录：`.sisyphus/evidence/`
- 推荐命名：`task-{编号}-{类型}-{说明}.txt`
  - 示例：
    - `task-01-plan-prd-mapping.txt`
    - `task-01-exec-red-log.txt`
    - `task-01-verify-gate-g3.txt`

## 3) 最低通过标准
- 每一步至少 1 份计划证据 + 1 份执行证据 + 1 份验证证据
- 任何一步缺少任一类证据，视为未完成

## 4) 真实性校验（反作弊）
- 交叉校验：计划目标 ↔ 实际 diff ↔ 测试日志
- 随机复跑：抽取步骤复跑测试与关键命令
- 越权抽检：随机检查权限路径与敏感操作
- 不接受截图替代原始文本日志

## 5) 失败处理
1. 标记失败证据
2. 记录失败 Gate
3. 回滚到最近可验证状态
4. 重新提交三类证据后再申请通过
