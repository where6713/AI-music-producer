# 核心业务逻辑清单（人工主导）

> 目标：明确哪些模块必须人工主导实现，避免把核心判断完全外包给自动生成。

## 1. 核心逻辑判定标准
满足任一条即归类为核心逻辑：
- 直接决定业务结论（推荐、判定、风险分级）
- 影响权限或安全边界
- 影响用户关键成本（如抽卡前决策）

## 2. 本项目核心逻辑范围

### A) 业务决策核心
- friction_calculator（摩擦度核心判定）
- lyric_architect（中文发音/声调/烂梗拦截规则）
- prompt_compiler（生成策略与约束拼装）

### B) 分析核心
- acoustic_analyst（关键声学特征计算）
- style_deconstructor（参考曲结构与要素抽取）

### C) 安全与自愈边界
- shell_probe 的允许/禁止命令策略
- py_eval 的执行隔离策略
- 异常自愈分流规则

## 3. 人工主导要求
- 核心逻辑的规则定义由人工确认
- 核心阈值变更需单独记录理由
- 核心模块改动必须附带审计复核结果

## 4. 非核心（可优先由 AI 辅助）
- 文档模板、清单、报告框架
- 通用脚本骨架（不含核心策略）
- 展示层与解释层文本

## 5. 通过判定
- 本轮改动已标注是否触及核心逻辑
- 触及时有人工确认记录
- 对应审计证据已归档
