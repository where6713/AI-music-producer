# 审计调度机制

> 目标：以固定频率 + 阈值触发方式，持续发现并处理安全漏洞、冗余代码、越权访问问题。

## 1. 触发条件

### 1.1 固定频率触发
- 周审计：每周一 10:00 执行一次全量审计（安全漏洞 + 冗余代码 + 越权访问）。
- 里程碑审计：每完成 3 个任务（以 `docs/governance/dev-board.md` 的 `Tasks Done` 增量计）执行一次结构审计。
- 合并前审计：任意分支准备合并到主线前必须执行一次综合审计。

### 1.2 风险分级触发
- 高风险（立即触发，T+0）：满足任一条件
  - 改动包含权限边界相关路径（鉴权、访问控制、敏感配置）。
  - 命中安全相关告警（密钥泄露、危险命令、未授权访问路径）。
  - 单次改动跨 4 个及以上目录。
- 中风险（24 小时内触发）：满足任一条件
  - 单次改动跨 2-3 个目录。
  - 出现重复脚本或重复规则定义迹象。
  - 测试通过率较上一轮下降 5% 及以上。
- 低风险（并入周审计）：不满足高风险和中风险条件。

## 2. 审计内容

### 2.1 安全漏洞审计
- 检查是否存在密钥、令牌、私钥等敏感信息暴露。
- 检查是否存在危险命令链路或可绕过约束执行路径。
- 检查是否存在未受控依赖变更导致的已知漏洞引入。

### 2.2 冗余代码审计
- 检查同职责脚本是否重复实现（同义不同名、重复流程）。
- 检查规则文档是否重复定义同一权威正文。
- 检查是否存在仅为通过测试而引入的低价值重复逻辑。

### 2.3 越权访问审计
- 检查访问控制是否被绕过，是否存在越权访问路径。
- 检查权限变更是否缺少审批记录或缺少最小权限约束。
- 检查核心模块是否存在跨边界调用且未声明授权关系。

## 3. 处置规则

### 3.1 分级判定
- 阻断项（Blocker）
  - 发现可利用的安全漏洞。
  - 发现可复现的越权访问。
  - 发现影响主流程正确性的高危冗余代码。
- 警告项（Warning）
  - 未直接造成风险但已偏离规范，如重复实现、边界模糊。
- 建议项（Advisory）
  - 优化类问题，不影响当前交付正确性和安全性。

### 3.2 处置时限
- 阻断项：立即停止进入下一 Gate，24 小时内完成修复并复审。
- 警告项：登记责任人和截止时间，5 个工作日内完成整改并复核。
- 建议项：纳入后续迭代清单，在下一个里程碑前确认是否采纳。

### 3.3 闭环要求
- 每次审计必须产出可复核记录，至少包含：触发条件、审计内容、发现分级、处置动作、复审结论。
- 未完成复审的阻断项视为审计失败，不得声明任务完成。
- 审计记录需同步到证据目录并可通过命令检索。
