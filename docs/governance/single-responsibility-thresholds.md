# 单一职责判据阈值

> 目的：把“单一职责”从原则变成可检查标准。

## 1. 文件级阈值（建议默认）
- 文档文件：单文件只承载一个主题（门禁、证据、审计等）
- 业务代码文件：单文件不混合多个业务域职责
- 脚本文件：单脚本只做一类检查或一类动作

## 2. 职责漂移检测信号
- 文件标题/说明出现两个以上独立目标
- 一次变更同时触发不相关模块
- 脚本同时承担“检查 + 业务处理”两类职责

## 3. 触发阈值（用于审计）
- 发现“同义不同名”脚本：立即标记漂移风险
- 发现同一主题多份权威定义：立即标记漂移风险
- 发现计划外跨域改动：进入审计复核

## 4. 处理策略
- 轻度漂移：拆分职责并补充说明
- 中度漂移：回滚并按计划重做
- 重度漂移：阻断合并，要求重新规划

## 5. 与现有规则的衔接
- 命名和目录边界以《目录框架规范.md》为准
- 文档归属与复用约束以 manifest + 一致性脚本为准
