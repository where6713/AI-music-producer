# ONE LAW

## 唯一法则

凡新增内容，必须同时满足四条：

1. 能追溯到 `docs/映月工厂_极简歌词工坊_PRD.json` 的明确条款。
2. 能被命令验证（lint、测试或脚本输出），禁止口头通过。
3. 不制造冗余维护面（不新增重复中间件、重复流程、重复数据资产）。
4. 不违背 PRD 红线：Single-Pass 优先、最多 1 次定向重写、输出三件套。

任一不满足：不落库、不合并、不放行。

## 开发顺序法则

先法则，后实现；先校验，后扩展；先最小可用，后优化。

- 法则冲突或来源不明：立即停止实现。
- 校验未定义：先补校验，再写实现。
- lint 或测试未通过：禁止新增复杂流程。

## 产品红线

### 红线 A：绝不造轮子

- 必须优先复用成熟依赖：`anthropic`、`pypinyin`、`jieba`、`rapidfuzz`、`pydantic`、`typer`、`rich`。
- 禁止引入“自研替代现成库”的工程炫技。

### 红线 B：绝不让模型重复做同一件事

- 默认 1 次 LLM 调用完成蒸馏、规划、填词、自检。
- 仅当 lint 失败时允许 1 次定向重写。
- 单次 run 最多 2 次 LLM 调用。

### 红线 C：绝不发明平台不识别概念

- Suno 标签仅允许白名单语法。
- 禁止自造元标签、私有协议标签、隐藏控制码。

### 红线 D：绝不回潮复杂中间件

- 永不重新引入 Wide Sampling、LLM-as-Judge、Motif Atlas/DMR/SVO 中间件、Web UI。

### 红线 E：绝不滑坡为复杂检索系统

- 禁止向量数据库与 embedding 模型。
- 禁止运行时语料清洗与 LLM 打标签。
- Few-Shot 示例数不得超过 3。
- 三变体必须来自一次调用内输出，不得拆成多次独立调用。

---

## 输出契约（唯一交付目标）

系统最终交付目标固定为：

- `out/lyrics.txt`
- `out/style.txt`
- `out/exclude.txt`

同时保留调试与追溯输出：

- `out/lyric_payload.json`
- `out/trace.json`

任何与该目标无关的流程扩张，默认不做。

---

## AI-Oriented 文档法（防漂移）

- 文档规范必须唯一归属，禁止多处复写同一规范正文。
- 归属关系由 `docs/ai_doc_manifest.json` 声明。
- 一致性校验通过 `tools/scripts/check_docs_consistency.ps1` 执行。

## 合并门槛

一次变更进入主线必须满足：

- [ ] 变更可映射到唯一 PRD 条款
- [ ] 关键路径有可执行验证命令
- [ ] 未引入 PRD 禁止项
- [ ] hooks 与 CI 口径一致
